# CLAUDE.md

Этот файл содержит инструкции для Claude Code (claude.ai/code) при работе с данным репозиторием.

## Обзор проекта

**Система управления веб-хостингом** для проведения семинаров по веб-разработке. Автоматизирует развёртывание изолированных учебных сред на общем сервере — каждый студент получает Unix-аккаунт, поддомен, базу данных MySQL и преднастроенный WordPress.

## Скрипты

| Скрипт | Запуск от | Использование |
|--------|-----------|---------------|
| `wsb2-install.sh` | root | Первоначальная настройка сервера — интерактивные вопросы конфигурируют стек |
| `wsb2-newstudent.sh` | teacher (sudo) | `wsb2-newstudent.sh <группа> <студент>` |
| `wsb2-delstudent.sh` | teacher (sudo) | `wsb2-delstudent.sh <студент>` — **необратимая операция** |

Два скрипта (`wsb2-tarbkp.sh`, `wsb2-zipbkp.sh`) не хранятся в репозитории — они скачиваются с GitHub во время установки.

## Архитектура

**Процесс установки**: `wsb2-install.sh` работает в интерактивном режиме, устанавливает программный стек, создаёт каталог `~/.wsb2/` для аккаунта преподавателя, затем через `sed` вписывает домен, пароли и выбор веб-сервера в вспомогательные скрипты и размещает их в `~/.wsb2/bin/`.

**Создание аккаунта студента** (`wsb2-newstudent.sh`):
1. Создаёт Unix-пользователя в `/home/{группа}/{студент}/`
2. Копирует WordPress из `~/.wsb2/src/wordpress/` преподавателя
3. Создаёт базу данных MySQL и пользователя с индивидуальными учётными данными
4. Записывает конфиг виртуального хоста в `/etc/nginx/sites-available/` или `/etc/apache2/sites-available/`
5. Перезагружает веб-сервер

**Схема URL**: преподаватель — `https://{домен}`, студенты — `https://{студент}.{домен}` (требуется wildcard DNS или отдельные записи для каждого поддомена).

**Конфигурация встроена в скрипты**: внешних конфигурационных файлов нет. `wsb2-install.sh` подставляет значения в вспомогательные скрипты через `sed` во время установки. Чтобы узнать текущие значения на развёрнутом сервере — читать переменные в начале `wsb2-newstudent.sh` и `wsb2-delstudent.sh`.

## Системные требования

- Debian GNU/Linux 12 (скрипты завершаются с ошибкой на других дистрибутивах)
- PHP 7.3 с FPM, MariaDB, ProFTPD, Memcached
- Веб-сервер: Nginx (вариант 1) или Apache2 (вариант 2)
- WordPress русской локали (`latest-ru_RU.zip`) скачивается во время установки

## Тестирование

Автоматических тестов, линтеров и сборки нет. Изменения проверяются вручную — развёртыванием на сервере Debian 12 и запуском скриптов.
