# Web Site Building 2.0

Система автоматизированного развёртывания учебных хостинг-сред для проведения семинаров по веб-разработке. Каждый студент получает изолированное окружение: Unix-аккаунт, поддомен, базу данных MySQL и преднастроенный WordPress — одной командой преподавателя.

## Возможности

- Автоматическая установка и настройка полного LAMP/LEMP-стека одним скриптом
- Выбор веб-сервера: **Nginx** или **Apache2**
- Создание учётной записи студента за секунды: Unix-пользователь, MySQL-база, виртуальный хост, WordPress
- Организация студентов по группам (`/home/{группа}/{студент}/`)
- WordPress на русском языке, преднастроенный и готовый к установке
- SSH и FTP-доступ для студентов
- phpMyAdmin для управления базами данных через браузер
- Откат изменений при ошибке создания аккаунта
- Скрипты резервного копирования (tar.gz и zip) с автоудалением бэкапов старше 14 дней
- Включение/выключение сайта студента без удаления данных

## Требования

| Компонент | Версия |
|-----------|--------|
| ОС | Debian GNU/Linux 12 или 13 |
| Веб-сервер | Nginx **или** Apache2 |
| PHP | 8.x с FPM (устанавливается автоматически) |
| СУБД | MariaDB (устанавливается автоматически) |
| Прочее | ProFTPD, Memcached, phpMyAdmin (устанавливаются автоматически) |

> **Важно:** Скрипты завершаются с ошибкой на дистрибутивах, отличных от Debian 12/13.

Для студентов необходим wildcard DNS-запись `*.ваш-домен` или отдельная A/CNAME-запись для каждого поддомена.

## Быстрый старт

Выполните от имени **root** на свежем сервере Debian 12/13:

```bash
apt update && apt install wget && wget https://raw.githubusercontent.com/koo2018/websitebuilding2/main/wsb2-install.sh && bash wsb2-install.sh
```

Скрипт задаст 5 вопросов и настроит весь стек.

## Установка

### Что спрашивает скрипт установки

1. **Веб-сервер** — Nginx (1) или Apache2 (2)
2. **Имя аккаунта преподавателя** — будет создан Unix-пользователь с sudo-привилегиями (по умолчанию: `teacher`)
3. **Домен сервера** — например, `example.ru`; преподаватель получит `https://teacher.example.ru`
4. **Пароль root для MariaDB**
5. **Общий пароль для баз данных студентов** — все студенты получат одинаковый пароль БД (и системный пароль тоже)

### Что делает скрипт установки

- Создаёт Unix-аккаунт преподавателя с sudo-правами
- Устанавливает: Nginx/Apache2, PHP-FPM, MariaDB, phpMyAdmin, ProFTPD, Memcached и утилиты
- Увеличивает лимиты PHP: `upload_max_filesize` и `post_max_size` до 64 МБ
- Скачивает WordPress (`latest-ru_RU.zip`) в `~/.wsb2/wordpress/`
- Скачивает вспомогательные скрипты (`wsb2-newstudent.sh`, `wsb2-delstudent.sh`, скрипты бэкапа) в `~/.wsb2/bin/` и встраивает в них домен, пароли и версию PHP через `sed`
- Добавляет `~/.wsb2/bin/` в `$PATH` преподавателя
- Настраивает виртуальный хост для сайта преподавателя и phpMyAdmin
- Устанавливает лимит подключений MariaDB до 200
- Перезагружает сервер

### После установки

Войдите заново под аккаунтом преподавателя. Команды `wsb2-newstudent.sh` и `wsb2-delstudent.sh` будут доступны без указания пути.

## Управление студентами

> Все команды выполняются от имени **преподавателя** (требуется sudo).

### Создать студента

```bash
wsb2-newstudent.sh <группа> <студент>
```

**Пример:**

```bash
wsb2-newstudent.sh web2024 ivanov
```

Что происходит:
1. Создаётся Unix-группа `web2024` (если её нет) и каталог `/home/web2024/`
2. Создаётся пользователь `ivanov` в `/home/web2024/ivanov/`
3. Устанавливается пароль (общий пароль БД студентов)
4. WordPress копируется в `/home/web2024/ivanov/www/` с правильными правами
5. Автоматически создаётся `wp-config.php` с параметрами подключения к БД
6. Создаётся MySQL-пользователь `ivanov` и база данных `ivanov`
7. Регистрируется виртуальный хост `ivanov.ваш-домен`
8. Перезагружается веб-сервер

Студент сразу может открыть `http://ivanov.ваш-домен` и запустить мастер установки WordPress.

> При ошибке на любом шаге выполняется автоматический откат: удаляется виртуальный хост, база данных и Unix-аккаунт.

### Удалить студента

```bash
wsb2-delstudent.sh <студент>
```

**Пример:**

```bash
wsb2-delstudent.sh ivanov
```

> **Внимание: операция необратима.** Удаляются виртуальный хост, Unix-аккаунт со всеми файлами (`--remove-home`) и MySQL-база данных студента.

### Временно выключить сайт студента

```bash
wsb2-offsite.sh <студент>
```

Отключает виртуальный хост и перезагружает веб-сервер. Unix-аккаунт, файлы и база данных **не затрагиваются**. Операция обратима.

### Включить сайт обратно

```bash
wsb2-onsite.sh <студент>
```

## Резервное копирование

После установки в `~/.wsb2/bin/` появляются два скрипта бэкапа. Запускаются вручную или через cron.

### `wsb2-tarbkp.sh` — бэкап в формате `.tgz`

Создаёт дамп всех баз данных MySQL и tar-архив каталога `/home/` (исключая домашний каталог преподавателя), упаковывает всё в один `.tgz`-файл с датой в имени.

### `wsb2-zipbkp.sh` — бэкап в формате `.zip`

Аналогично, но создаёт единый `.zip`-архив с дампом БД и каталогом `/home/`.

Оба скрипта:
- Хранят бэкапы в `~/backups/` (настраивается при установке)
- Автоматически удаляют файлы старше **14 дней**

**Пример записи в crontab** (ежедневно в 3:00):

```bash
0 3 * * * /home/teacher/.wsb2/bin/wsb2-tarbkp.sh
```

## Архитектура

```
Интернет
    │
    ▼
[ Веб-сервер: Nginx или Apache2 ]
    │
    ├── teacher.example.ru  →  /home/teacher/.wsb2/www/  (сайт преподавателя + phpMyAdmin)
    ├── ivanov.example.ru   →  /home/web2024/ivanov/www/ (WordPress студента)
    ├── petrov.example.ru   →  /home/web2024/petrov/www/
    └── ...
    │
    ▼
[ PHP-FPM (www-data) ]
    │
    ▼
[ MariaDB ]
    ├── ivanov  (база данных студента)
    ├── petrov
    └── phpmyadmin
```

### Схема URL

| Сайт | Адрес |
|------|-------|
| Преподаватель | `https://teacher.example.ru` |
| Студент | `https://{студент}.{домен}` |
| phpMyAdmin | `https://teacher.example.ru/phpmyadmin` |

## Структура каталогов

```
/home/
├── teacher/                  # домашний каталог преподавателя
│   ├── .wsb2/
│   │   ├── bin/              # скрипты управления (добавлены в $PATH)
│   │   │   ├── wsb2-newstudent.sh
│   │   │   ├── wsb2-delstudent.sh
│   │   │   ├── wsb2-tarbkp.sh
│   │   │   └── wsb2-zipbkp.sh
│   │   ├── src/
│   │   │   └── wordpress/    # эталонная копия WordPress для клонирования
│   │   └── www/              # веб-корень сайта преподавателя
│   ├── .log/                 # логи доступа и ошибок
│   └── backups/              # резервные копии (создаётся при первом бэкапе)
│
└── web2024/                  # группа студентов
    ├── ivanov/
    │   ├── www/              # WordPress сайта ivanov.example.ru
    │   └── .log/             # логи студента
    └── petrov/
        ├── www/
        └── .log/
```

**Конфигурация встроена в скрипты** — при установке `wsb2-install.sh` записывает домен, пароли и версию PHP напрямую в вспомогательные скрипты через `sed`. Внешних конфигурационных файлов нет. Чтобы узнать текущие параметры развёрнутого сервера — смотрите переменные в начале `wsb2-newstudent.sh`.

## Права доступа к файлам

WordPress работает с `FS_METHOD = direct` — PHP пишет файлы напрямую, без FTP-прослойки. PHP-FPM работает от пользователя `www-data`, поэтому группа должна иметь права записи на `wp-content`.

| Путь | Владелец | Права |
|------|----------|-------|
| `/home/{группа}/{студент}/` | `{студент}:{группа}` | `711` |
| `/home/{группа}/{студент}/www/` | `{студент}:www-data` | `755` |
| Файлы в `www/` | `{студент}:www-data` | `644` |
| `wp-content/` и поддиректории | `{студент}:www-data` | `755` + `g+w` |
| `wp-content/uploads/` | `{студент}:www-data` | `g+w` рекурсивно |
| `wp-config.php` | `{студент}:www-data` | `640` |

Если WordPress жалуется «не удалось скопировать файл» при установке плагинов или тем — проверьте права на `wp-content/`.

## Известные особенности

**Ошибки `Can't connect to local MySQL server` в логе установки** — штатное поведение. При `apt install phpmyadmin` пакет пытается настроить базу данных до того, как MariaDB запущена. Скрипт повторно настраивает phpMyAdmin после старта MariaDB — это нормально, на работу системы не влияет.

**Общий пароль студентов** — все студенты получают одинаковый системный пароль и пароль к MySQL-базе. Это сознательное упрощение для учебной среды.

**Студент не может сменить системный пароль** — смена пароля через `passwd` не меняет пароль в WordPress. В учебных целях это несущественно.

**Нет HTTPS из коробки** — виртуальные хосты настраиваются на HTTP (порт 80). Для HTTPS настройте Let's Encrypt / Certbot отдельно после установки.
